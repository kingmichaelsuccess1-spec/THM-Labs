## Description
This room provides hands-on familiarity with the core function of a Tier 1 SOC analyst: alert triage. It covers the structure and lifecycle of a SOC alert, how to interpret alert fields and statuses, and how to classify alerts accurately. Practice was conducted using a simulated SIEM/dashboard environment with real-world style alerts.

## Key Concepts Covered
- The concept of a SOC alert and how it fits the detection pipeline
- Alert fields: source, timestamp, severity, indicators
- Alert statuses: new, in progress, escalated, closed
- Alert classification: True Positive, False Positive, Benign
- Severity level assessment
- L1 triage methodology — step-by-step alert review process
- SOC workflows practiced via simulated SIEM/dashboard environment

## SOC Analyst Takeaways
- Triage: Reviewing alert fields to determine validity and urgency
- Classification: Applying TP/FP/Benign labels and severity levels accurately
- Workflow: Structured SOC process from alert intake to resolution or escalation
- Tooling: Navigating a SIEM-style dashboard for alert investigation

## Skills Demonstrated
- Alert field interpretation and status tracking
- Alert classification (TP/FP/Benign) and severity assessment
- L1-level triage decision-making
- SIEM/dashboard navigation for SOC workflows
